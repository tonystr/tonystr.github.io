Welcome / hello & ATIRAG

Feb 23: 2018


Hello, everyone! This is the start of a devlog, which is practically the same as a blog, but focused on game/software development. The purpose of this devlog is to force myself to work. I will be posting weekly updates on whatever I'm working on, and hopefully some technical details. Each post will have atleast one image attached, even if unrelated.

This week, I've been working on this subpage (the devlog), and I've started working on a simple 2d bullethell shoot 'em up. After 5 years of programming, I've still not actually released any games. My goal is to get this done within a month (Release date: 20th March 2018), so it'll be faily simplistic. I do however have to spend some of that time on school and in other languages, but with a solid goal, I can't go wrong.

Here are some gifs of a very crude prototype

ATIRAG 2018

ATIRAG 2018

This enemy type uses a "dna" variable which decides what behaviours the enemy has. Such as: does it move horizontally? How many bullets does it shoot? How fast is it? - and more to come.

The stars in the background move at different speeds, and have their color adjusted accordingly, that way it shows depth and looks 3d. This doesn't show very well in the gif, but maybe I'll post an mp4 later, when I eventually change it to look even better. Another trick with the stars, is that I'm actually looping the same stars over and over again. The amount of stars on screen is always the same because of this. This works quite well, because the stars move at different speeds, so the player won't ever notice the pattern is looping, however, because I can, whenever a star hits the bottom of the screen, it changes it's x coordinate and speed, to appear as a different star next time.

Right now, this game looks boring, plays boring, and lacks both polish and content. Therefore, I've compiled a list of features I'm thinking of adding or changing.

  • Colored stars
  • Larger bullets
  • More challenging and smarter AI
  • Bullet formations
  • Bullet upgrades
  • Visual effects upon death (animation/particles)
  • Sounds

There are of course more things I want to add, but these are my priority right now. I need a solid skeleton before I start adding the flesh. Hopefully some of these changes will be present in the next post.
