Motherlode begins

27. Mar 2018


I've done two interesting things this week after releasing ATIRAG. First of all, I wrote a script in javascript that takes either a number or name as input, and outputs a link, image, title and some more information from a list of pixelart tutorials hosted by studio-miniboss. Basically, it's a script for embeding pixelart tutorials by pedro/saint11 with a Discord bot. The script sends a http request to blog.studiominiboss.com/pixelart, retrieves the whole html document as a string, then uses the power of RegEx to search the website against the input. It then uses some more regex to find the rest of the information, then some more, and just a little bit more RegEx to hold it all together. I'd love to embed a live demo of it right here, however it's built in node.js and requires the http package, and I'm not comfortable with XMLHttpRequests yet.

Motherlode 2018

About 8 years ago, I played a game called Motherload. I played the game again, a couple days ago, and was fascinated by how well balanced it was. Out of love for the game, I decided to try to recreate it, maybe with some of my own features.

Currently, I've got the basic movement down and the level generates a grid of blocks, some of which hold ore. The blocks autotile and can be mined through in a similar fashion to the original Motherload. The background linearly interpolates between a set of colors as you traverse the world vertically.

Right now, I'm working on adding an inventory to the game, and some sprites for the explosions (dynamite and plastics).
Here's the planned-features list for Motherlode:

  • Item Sprites
  • Grid-based inventory
  • Gas station + fuel system
  • Junk shop + upgrade system
  • Mineral processing + money/points system
  • Emendation station + hull system
  • Terrain breaking while mining
  • Saving / loading
  • Main Menu
  • Satan?
  • GUI
